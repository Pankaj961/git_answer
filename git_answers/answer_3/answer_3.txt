### Branch Creation and Feature Development

First, I created two new branches, `master_1` and `master_2`, from the `main` branch.

```bash
git checkout main
git checkout -b master_1
git checkout main
git checkout -b master_2
```

In the `master_1` branch, I created a **To-Do Page** in the frontend. The page contains a form with two input fields: **Item Name** and **Item Description**.

After completing the frontend work, I saved the changes and committed them.

```bash
git add .
git commit -m "Added To-Do page with item form"
```

Next, I switched to the `master_2` branch and created a backend route named `/submittodoitem`. This route accepts `itemName` and `itemDescription` through a **POST** request and stores the data in the MongoDB database.

After implementing the backend feature, I committed the changes.

```bash
git add .
git commit -m "Added /submittodoitem API with MongoDB integration"
```

Finally, I switched back to the `main` branch and merged both branches one by one.

```bash
git checkout main
git merge master_1
git merge master_2
git push origin main
```

This completed the assignment by creating separate branches for the frontend and backend, implementing the required features, and merging both branches into the `main` branch.
