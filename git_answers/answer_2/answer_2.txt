### Git Branch and Merge Assignment

First, I created a new branch named `Tutedude_new` and switched to it.

```bash
git checkout -b Tutedude_new
```

In this branch, I updated the content of the JSON file that is used by the `/api` route in the Flask project. After making the changes, I added the updated file to the staging area and committed the changes.

```bash
git add .
git commit -m "Updated JSON data for API route"
```

Next, I switched back to the `main` branch and merged the `Tutedude_new` branch into it.

```bash
git checkout main
git merge Tutedude_new
```

If any merge conflicts appeared, I resolved them by keeping the changes from the `Tutedude_new` branch. After resolving the conflicts, I added the resolved files to the staging area, committed the merge, and pushed the updated code to the remote repository.

```bash
git add .
git commit -m "Resolved merge conflicts and merged Tutedude_new into main"
git push origin main
```

This completed the process of updating the JSON file in a separate branch, merging it into the `main` branch, resolving any conflicts, and pushing the final changes to GitHub.
